## 🔧 Требования

- PHP 8.0 или выше
- SQLite3 с поддержкой PDO
- Composer
- PHPUnit 10.0 или выше
- Расширения PHP:
  - `pdo`
  - `pdo_sqlite`
  - `mbstring`
  - `json`
  - `xml`

## 📦 Установка

### 1. Установка зависимостей

Перейдите в директорию `backend/` и выполните:

```bash
cd backend
php composer.phar install
```

Или, если Composer установлен глобально:

```bash
composer install
```

### 2. Инициализация тестовой базы данных

Тестовая база данных создается автоматически при первом запуске тестов. Файл `test-database.db` будет создан в директории `backend/`.

### 3. Проверка конфигурации

Убедитесь, что файл `phpunit.xml` настроен правильно и указывает на правильные пути.

## 📁 Структура тестов

```
backend/tests/
├── Unit/                          # Модульные тесты
│   ├── PhoneValidationTest.php    # Тесты валидации телефона (10 тестов)
│   └── PasswordValidationTest.php # Тесты валидации пароля (10 тестов)
├── Functional/                    # Функциональные тесты
│   ├── Registration/
│   │   └── RegistrationTest.php   # Тесты регистрации (15 тестов)
│   ├── Login/
│   │   └── LoginTest.php          # Тесты входа (10 тестов)
│   └── Recovery/
│       └── RecoveryTest.php       # Тесты восстановления (15 тестов)
├── TestCase.php                   # Базовый класс для тестов
├── bootstrap.php                  # Инициализация тестового окружения
└── README.md                      # Документация
```

## 🚀 Запуск тестов

### Через командную строку

#### Запуск всех тестов:
```bash
cd backend
vendor/bin/phpunit
```

#### Запуск только модульных тестов:
```bash
vendor/bin/phpunit --testsuite "Unit Tests"
```

#### Запуск только функциональных тестов:
```bash
vendor/bin/phpunit --testsuite "Functional Tests"
```

#### Запуск конкретного теста:
```bash
vendor/bin/phpunit tests/Unit/PhoneValidationTest.php
```

#### Запуск с подробным выводом:
```bash
vendor/bin/phpunit --verbose
```

#### Запуск с покрытием кода:
```bash
vendor/bin/phpunit --coverage-html coverage/
```

### Через веб-интерфейс

1. Откройте в браузере: `http://localhost:8000/backend/test_results.php`
2. Нажмите кнопку "Запустить все тесты"
3. Дождитесь завершения выполнения
4. Просмотрите результаты

## 📝 Документация по тестам

### Изоляция тестов

Каждый тест работает с изолированной копией базы данных:
- Перед каждым тестом создается чистая БД
- После каждого теста данные очищаются
- Тесты не зависят друг от друга

### Таймауты

- Максимальное время выполнения одного теста: 1 секунда
- Медленные тесты автоматически помечаются как "SKIPPED"

### Обработка ошибок

Все ошибки обрабатываются с понятными сообщениями на русском языке.

### Безопасность

- Использование подготовленных SQL-запросов
- Валидация всех входных данных
- Защита от SQL-инъекций
- Хэширование паролей и ответов

## 🐛 Устранение неполадок

### Ошибка: "Class 'PHPUnit\Framework\TestCase' not found"
**Решение:** Установите зависимости через Composer:
```bash
composer install
```

### Ошибка: "PDOException: could not find driver"
**Решение:** Убедитесь, что расширение `pdo_sqlite` установлено:
```bash
php -m | grep pdo_sqlite
```

### Ошибка: "Permission denied" при создании БД
**Решение:** Проверьте права доступа к директории `backend/`:
```bash
chmod 755 backend/
```

### Тесты выполняются слишком долго
**Решение:** Проверьте настройки таймаута в `phpunit.xml` и убедитесь, что тестовая БД не слишком большая.

## 🔄 Обновление тестов

При добавлении новых тестов:
1. Создайте файл теста в соответствующей директории
2. Наследуйтесь от класса `TestCase`
3. Используйте методы `setUp()` и `tearDown()` для инициализации
4. Запустите тесты для проверки
