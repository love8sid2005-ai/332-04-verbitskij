# Как работают тесты и где они отображаются

## 🎯 Где находится веб-интерфейс тестов

Веб-интерфейс для запуска и просмотра тестов находится по адресу:

```
http://localhost:8000/backend/test_results.php
```

## 🚀 Как запустить веб-интерфейс

### Шаг 1: Запустите PHP сервер

Откройте терминал в корне проекта и выполните:

```bash
php -S localhost:8000
```

### Шаг 2: Откройте в браузере

Перейдите по адресу:
```
http://localhost:8000/backend/test_results.php
```

## 🔄 Процесс выполнения теста

### 1. Инициализация (bootstrap.php)

Когда PHPUnit запускается, он загружает `bootstrap.php`, который:

```php
// 1. Определяет путь к тестовой БД
define('TEST_DB_PATH', __DIR__ . '/../test-database.db');
define('DB_PATH', TEST_DB_PATH);

// 2. Подключает автозагрузчик Composer
require_once __DIR__ . '/../vendor/autoload.php';

// 3. Подключает config.php (функции валидации и т.д.)
require_once __DIR__ . '/../config.php';
```

### 2. Запуск теста (TestCase.php)

Каждый тест наследуется от `TestCase`, который:

```php
protected function setUp(): void {
    // 1. Создает чистую тестовую БД
    $this->initTestDatabase();
    
    // 2. Подключается к БД
    $this->db = getTestDB();
}

protected function tearDown(): void {
    // Очищает данные после теста
    $this->cleanupTestData();
}
```

### 3. Выполнение теста

Например, тест регистрации:

```php
public function testSuccessfulRegistration(): void {
    // 1. Подготовка данных
    $phone = '+7-999-123-45-67';
    $password = 'Test1234';
    
    // 2. Выполнение (симуляция регистрации)
    $normalizedPhone = normalizePhone($phone);
    $hashedPassword = password_hash($password, PASSWORD_DEFAULT);
    
    // 3. Сохранение в БД
    $stmt = $this->db->prepare("INSERT INTO users ...");
    $stmt->execute([...]);
    
    // 4. Проверка результата
    $this->assertTrue($result);
    $this->assertEquals(1, $this->getUserCount());
}
```

### 4. Сохранение результатов

PHPUnit сохраняет результаты в `test-results.xml`:

```xml
<testsuite name="RegistrationTest" tests="15" failures="0" errors="0">
    <testcase name="testSuccessfulRegistration" class="RegistrationTest" time="0.023"/>
    <testcase name="testRegistrationDuplicatePhone" class="RegistrationTest" time="0.015"/>
    ...
</testsuite>
```

### 5. Отображение в веб-интерфейсе

`test_runner.php` парсит XML и возвращает JSON:

```json
{
    "success": true,
    "total": 60,
    "passed": 58,
    "failed": 2,
    "skipped": 0,
    "time": 2.45,
    "groups": [
        {
            "name": "RegistrationTest",
            "tests": [
                {
                    "name": "testSuccessfulRegistration",
                    "status": "passed",
                    "time": 0.023,
                    "group": "Functional / Registration"
                }
            ]
        }
    ]
}
```

JavaScript в `test_results.php` получает эти данные и отображает их в таблице.

## 🎨 Что отображается в веб-интерфейсе

### Панель управления

- **🚀 Запустить все тесты** - запускает все 60 тестов
- **🔍 Модульные тесты** - только тесты валидации (20 тестов)
- **⚙️ Функциональные тесты** - тесты регистрации, входа, восстановления (40 тестов)
- **🗑️ Очистить результаты** - скрывает результаты

### Статистика

После запуска тестов отображается:

- **Всего тестов** - общее количество
- **Успешных** - зеленым цветом ✅
- **Неудачных** - красным цветом ❌
- **Пропущенных** - желтым цветом ⚠️
- **Время выполнения** - в секундах
- **Успешность** - процент успешных тестов

### Таблица результатов

Для каждого теста показывается:

| Название теста | Статус | Время | Группа |
|---------------|--------|-------|--------|
| testValidPhoneFormat | ✅ passed | 0.012s | Unit / PhoneValidation |
| testInvalidPhone | ❌ failed | 0.008s | Unit / PhoneValidation |

Если тест провалился, показывается детальное сообщение об ошибке.

### Экспорт результатов

- **📥 Экспорт JSON** - скачивает результаты в формате JSON
- **📄 Экспорт HTML** - скачивает всю страницу с результатами

## 🐛 Решение проблем

### Тесты не запускаются через веб-интерфейс

1. **Проверьте, что PHPUnit установлен:**
   ```bash
   cd backend
   vendor/bin/phpunit --version
   ```

2. **Проверьте права доступа:**
   - Убедитесь, что PHP может выполнять команды
   - Проверьте права на запись в папку `backend/`

3. **Проверьте консоль браузера (F12):**
   - Откройте вкладку "Console"
   - Посмотрите на ошибки JavaScript

4. **Проверьте логи PHP:**
   - Включите отображение ошибок в `php.ini`
   - Или проверьте логи веб-сервера

### Тесты выполняются слишком долго

- Проверьте, что тестовая БД не слишком большая
- Убедитесь, что нет зависших процессов PHP
- Проверьте настройки таймаута в `phpunit.xml`
