# Портфолио с системой аутентификации

Персональное портфолио с безопасной системой аутентификации по номеру телефона. Проект разработан в учебных целях и демонстрирует применение современных практик веб-разработки.

## 📋 Описание функциональности

### Основные возможности

- **Регистрация пользователя** - создание нового аккаунта с номером телефона и паролем
- **Авторизация** - вход в систему с проверкой учетных данных
- **Восстановление доступа** - двухэтапное восстановление пароля через секретный вопрос
- **Защищенное портфолио** - главная страница доступна только авторизованным пользователям
- **Выход из системы** - безопасное завершение сессии

### Поток пользователя

1. Пользователь переходит на сайт
2. Если не авторизован → редирект на страницу входа
3. Может зарегистрироваться или войти через существующий аккаунт
4. После успешной авторизации → доступ к личному портфолио
5. Может восстановить пароль, если его забыл
6. Может выйти из системы в любой момент

## 🚀 Инструкция по установке и запуску

### Требования

- PHP 7.2+
- Node.js (для npm)
- SQLite3 (встроена в PHP)

### Установка

1. Клонируйте репозиторий или распакуйте архив:
```bash
cd portfolio
```

2. Убедитесь, что установлены зависимости (только npm):
```fish
npm install
```

### Запуск

Для запуска приложения используйте команду:

```fish
npm start
```

Приложение запустится на `http://localhost:8000`

### Первый запуск

При первом запуске автоматически создается база данных `backend/database.db` с необходимой структурой.

### Тестовые учетные данные

После регистрации вы можете использовать следующие данные для входа:

- **Номер телефона**: +7 9221110500 (вводить: 9221110500)
- **Пароль**: Test123

### Доступные страницы

| Страница | URL | Описание |

## 🗄️ Описание структуры базы данных

### Таблица users

Хранит информацию о зарегистрированных пользователях.

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    phone TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    secret_question TEXT NOT NULL,
    secret_answer TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

### Поля таблицы

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | INTEGER | Уникальный идентификатор пользователя (автоинкремент) |
| `phone` | TEXT | Номер телефона в формате +7XXXXXXXXXX (уникальный) |
| `password` | TEXT | Хэшированный пароль (password_hash) |
| `secret_question` | TEXT | Секретный вопрос для восстановления пароля |
| `secret_answer` | TEXT | Хэшированный ответ на секретный вопрос |
| `created_at` | DATETIME | Дата и время регистрации пользователя |

### Доступные секретные вопросы

1. Как зовут вашего первого домашнего питомца?
2. В каком городе вы родились?
3. Какой ваш любимый цвет?

## 🔒 Меры безопасности

### Защита от SQL-инъекций

- **Подготовленные запросы (Prepared Statements)**: Все SQL-запросы используют placeholders (?) вместо прямого вставления данных
- **Параметризованные запросы через PDO**: Использование методов `prepare()` и `execute()` с массивом параметров

Пример безопасного запроса:
```php
$stmt = $db->prepare("SELECT id FROM users WHERE phone = ?");
$stmt->execute([$phone]);
```

### Хэширование паролей

- **Функция password_hash()**: Использование PHP-встроенной функции с алгоритмом bcrypt
- **Безопасное сравнение**: Функция `password_verify()` защищает от timing-атак
- Пароли никогда не хранятся в открытом виде

Пример хэширования:
```php
$hashed = password_hash($password, PASSWORD_DEFAULT);
$is_valid = password_verify($password, $hashed);
```

### Валидация входных данных

- **Серверная валидация**: Все данные проверяются на сервере, не полагаемся на клиента
- **Валидация номера телефона**: Проверка формата +7XXXXXXXXXX (11 символов, 10 цифр после +7)
- **Валидация пароля**: Минимум 6, максимум 10 символов
- **Санитизация данных**: Использование `htmlspecialchars()` при выводе в HTML

### Защита от XSS (Cross-Site Scripting)

- **htmlspecialchars()**: Все пользовательские данные экранируются перед выводом в HTML
- **Экранирование в шаблонах**: `<?= htmlspecialchars($data) ?>`

### Управление сессиями

- **session_start()**: Инициализация сессии на каждом запросе
- **checkAuth()**: Проверка наличия валидной сессии перед доступом к защищенным страницам
- **session_destroy()**: Полное очищение сессии при выходе
- **Редирект неавторизованных пользователей**: Автоматический редирект на страницу входа

### Безопасное хранение секретных ответов

- **Хэширование ответов**: Ответы на секретные вопросы хэшируются так же, как пароли
- **Нечувствительность к регистру**: Ответы приводятся к нижнему регистру перед хэшированием
- Защита от перебора: Использование bcrypt делает перебор практически невозможным

### Защита от CSRF (Cross-Site Request Forgery)

- **POST запросы**: Все действия используют POST вместо GET
- **Проверка метода запроса**: `$_SERVER["REQUEST_METHOD"] === "POST"`

### Обработка ошибок

- **Общие сообщения об ошибках**: Пользователю показываются общие сообщения ("Пользователь не найден")
- **Логирование на сервере**: Детальные ошибки логируются, но не показываются пользователю
- **Защита от информационных утечек**: Различные сценарии (несуществующий пользователь, неверный пароль) показывают одинаковое сообщение
