# 📋 Описание функциональности

## Основные возможности

* **Регистрация пользователя** с номером телефона и паролем
* **Авторизация** с проверкой учётных данных
* **Восстановление доступа** через секретный вопрос
* **Защищённое портфолио**, доступное только авторизованным пользователям
* **Выход из системы** и безопасное завершение сессии

## Поток пользователя

1. Переходит на сайт
2. При отсутствии авторизации → перенаправление на страницу логина
3. Может войти или зарегистрироваться
4. После входа попадает в личное портфолио
5. При необходимости может восстановить пароль
6. Может выйти в любой момент

# 🚀 Установка и запуск

## Требования

* PHP 7.2+
* SQLite3
* Node.js (опционально, для `npm start`)

## Установка

```bash
cd portfolio
npm install   # опционально
```

## Запуск

Через npm:

```bash
npm start
```

Или напрямую:

```bash
php -S localhost:8000 -t public
```

Приложение будет доступно по адресу:
`http://localhost:8000`

## Первый запуск

При первом обращении автоматически создаётся файл базы данных:

```
backend/database.db
```

# 🧪 Тестирование

Во втором уроке создана система PHP-тестов с веб-интерфейсом.

## Структура тестов

```
tests/
├── ValidationTests.php
├── RegistrationTests.php
├── LoginTests.php
└── RecoveryTests.php

public/
└── test_results.php
```

## Запуск тестов

1. Запустить сервер:

```bash
php -S localhost:8000 -t public
```

2. Перейти по адресу:

```
http://localhost:8000/test_results.php
```

3. Выбрать категорию → нажать «Запустить тесты»

## Что проверяют тесты

### **Валидация**

* Формат номера телефона
* Длина пароля
* Пустые значения
* Неверные форматы

### **Регистрация**

* Создание нового пользователя
* Повторная регистрация с тем же номером
* Хеширование пароля
* Сохранение вопроса и ответа

### **Авторизация**

* Корректный вход
* Неверный пароль
* Ошибка при входе несуществующим пользователем
* Установка сессии

### **Восстановление**

* Поиск пользователя
* Вывод вопроса
* Проверка ответа
* Смена пароля

## Тестовая база данных

Тесты используют отдельную базу:

```
backend/test_database.db
```

Она создаётся автоматически и не влияет на основную.

# 🗄️ Структура базы данных

## Таблица `users`

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    phone TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    secret_question TEXT NOT NULL,
    secret_answer TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## Поля таблицы

| Поле            | Описание         |
| --------------- | ---------------- |
| id              | Уникальный ID    |
| phone           | Номер телефона   |
| password        | Хеш пароля       |
| secret_question | Секретный вопрос |
| secret_answer   | Хеш ответа       |
| created_at      | Дата регистрации |

## Секретные вопросы

1. Как зовут вашего первого домашнего питомца?
2. В каком городе вы родились?
3. Какой ваш любимый цвет?

# 🔒 Меры безопасности

* Подготовленные SQL-запросы (`prepare`, `execute`)
* Хеширование паролей (`password_hash`)
* Проверка паролей (`password_verify`)
* Хеширование секретного ответа
* Серверная валидация всех полей
* Экранирование HTML (`htmlspecialchars`)
* Управление сессиями (`session_start`, `session_destroy`)
* Все действия выполняются через POST
* Сообщения об ошибках не раскрывают лишних данных
